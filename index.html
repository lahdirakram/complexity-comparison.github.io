Partie I : Développement de l’algorithme et du programme du problème de la somme des n premiers nombres entiers naturels

1/- les algorithme :

  • En utilisant la boucle POUR :
    Var
	n,i,somme : entier;
Debut 
	lire(n);
	somme=0;
	pour i=0 a i<=n faire
		somme = somme + i;
	fait;
	affiche(somme);
Fin
    La complexcite de cet algorithme est : o(n)
  • En utilisant la boucle Tant que :
    Var
	n,i,somme : entier;
Debut 
	lire(n);
	somme=0;
	i=0;
	tantque i<=n faire
		somme = somme + i;
		i++;
	fait;
	affiche(somme);
Fin
    La complexcite de cet algorithme est : o(n)
  • En utilisant la boucle Répéter :
    Var
	n,i,somme : entier;
Debut
    
	lire(n);
	somme=0;
	i=0;
	repeter
		somme = somme + i;
		i++;
	jusque( i>n)
	affiche(somme);
Fin
    La complexcite de cet algorithme est : o(n)

2/- Les programmes itératifs correspondants avec le langage C:

  • Avec Boucle Pour:
    unsigned long long int forSum(unsigned long long int n){
	unsigned long long int somme=0,i;
	for(i=0;i<=n;i++){
		somme=somme+i;
	}
	return somme;
}
  • Avec Boucle Tant que :
    unsigned long long int whileSum(unsigned long long int n){
	unsigned long long int somme=0,i=0;
	somme=0;
	while(i<=n){
		somme=somme+i;
		i++;
	}
	return somme;
}
  • Avec Boucle Répéter :
    unsigned long long int doSum(unsigned long long int n){
	unsigned long long int somme=0,i=0;
	somme=0;
	do{
		somme=somme+i;
		i++;
	}while(i<=n);
	return somme;
}

3/- L'algorithme récursive :

  • fonction recsum (n : entier) 
Debut
	si n == 0 alors retourné 0
	si non retourne n+ rec(n-1)
Fin
    La complexcite de cet algorithme est : o(n)

4/- Le programe C correspondant: 

  • unsigned long long int recSum(unsigned long long int n){
	if(n == 0){
		return 0;
	}else{
		return (n+recSum(n-1));
	}
}
    REMARQUE: L'execution de ce programme est limité a des petites valeur de n car il faut augmenter la taille de la pile
